# Issues และ Bugs

## Issue #1: การกินสามต่อแล้ววกกลับยังทำไม่ได้

**สถานะ:** Closed ✅
**ความสำคัญ:** High

**รายละเอียด:**
KING ควรสามารถกินหลายตัวต่อเนื่อง (multi-capture) และสามารถ**วกกลับ**กินตัวที่อยู่ด้านหลังได้

**ตัวอย่าง:**
สมมุติ:
- KING สีขาว อยู่ที่ช่อง **1**
- เบี้ยสีดำ อยู่ที่ช่อง **15, 23, 16**

KING ควรกินได้หลายเส้นทาง:
1. `1-19` (กิน 15), `19-26` (กิน 23), `26-12` (กิน 16)
2. `1-19` (กิน 15), `19-12` (กิน 16), `12-26` (กิน 23) ← **วกกลับ!**

**การทำซ้ำ:**
1. จัดตำแหน่งตามตัวอย่างข้างบน
2. เลือก KING ที่ช่อง 1
3. ตรวจสอบว่ามี moves ที่สามารถกิน 3 ตัวในเส้นทางต่างกันหรือไม่

**แก้ไขที่คาดหวัง:**
KING สามารถกินได้ทุกทิศทาง (4 ทแยง) ในระหว่าง multi-capture รวมถึงการวกกลับกินด้วย

**สถานะปัจจุบัน:**
✅ **แก้ไขแล้ว!**

**สาเหตุ:**
ฟังก์ชัน `_get_king_capture_moves()` เมื่อมองหาศัตรูในแนวทแยง ถ้าเจอตัวที่ถูกกินไปแล้ว (อยู่ใน `captured_so_far`) มันจะ `break` ทันที แทนที่จะ `continue` ข้ามไปมองต่อ

**การแก้ไข:**
เปลี่ยนจาก:
```python
else:
    break  # เจอตัวเองหรือตัวที่กินไปแล้ว
```

เป็น:
```python
elif (check_row, check_col) in captured_so_far:
    continue  # เจอตัวที่กินไปแล้ว - ข้ามไปมองต่อ
else:
    break  # เจอตัวเอง - หยุด
```

**ไฟล์ที่แก้:** `game_logic.py:202-207`
